# 目標
Codex での開発運用に使う Skill と Multi-Agent の候補を、短い名前と簡潔な説明で整理する。

# Skill 一覧

- `plan`  
  要件を整理し、実装前の計画を作る

- `scan`  
  関連ファイルや影響範囲を調べる

- `implement`  
  計画で確定した範囲を実装する

- `test`  
  テストの追加・更新・実行を行う

- `review`  
  差分を確認し、問題点を洗い出す

- `docs`  
  README や仕様書などの文書を更新する

- `match`  
  既存コードの書き方や流儀に合わせる

- `refactor`  
  小さく安全な範囲で整理・改善する

# Multi-Agent 一覧

- `main`  
  要件整理、計画確定、最終実装、結果統合を担当する中心エージェント

- `explorer`  
  関連ファイルや既存実装パターンを調べる

- `tester`  
  既存テストを確認し、不足している観点を洗い出す

- `triager`  
  ログや再現条件を整理し、不具合の切り分けを行う

- `reviewer`  
  実装後の差分を見て、見落としやリスクを確認する

- `refactor`  
  安全に整理できる改善候補を探す

- `deps`  
  依存関係や設定変更の影響を確認する

# 運用方針

## 新規開発
1. **Skill `plan`** を使って、要件整理・実装方針・作業手順を作成する  
2. **Agent `explorer`** を使って、関連ファイルや既存実装パターンを調査する  
3. 必要に応じて **Agent `deps`** を使って、依存関係や設定変更の影響を確認する  
4. 必要に応じて **Agent `tester`** を使って、先に検証観点や既存テストの有無を確認する  
5. **Agent `main`** を使って、**Skill `implement`** により新規実装を進める  
6. 実装後に **Skill `test`** を使って、テストの追加・更新・実行を行う  
7. 実装後に **Agent `reviewer`** を使って差分を確認し、必要に応じて **Skill `review`** を使って問題点を洗い出す  
8. 作業完了後に **Skill `docs`** を使って、README や仕様書などの関連文書を更新する  

## 機能追加
1. **Skill `plan`** を使って、変更対象・影響範囲・実装手順を整理する  
2. **Skill `scan`** を使って、関連ファイルや変更が必要な箇所を洗い出す  
3. **Skill `match`** を使って、既存コードの命名・構造・実装パターンを確認する  
4. 必要に応じて **Agent `explorer`** を使って、既存実装の流れや周辺構造を詳しく調査する  
5. 必要に応じて **Agent `deps`** を使って、依存関係や設定変更の有無を確認する  
6. **Agent `main`** を使って、**Skill `implement`** により機能追加を行う  
7. 実装後に **Skill `test`** を使って、既存機能への影響確認と追加機能の検証を行う  
8. 実装後に **Agent `reviewer`** を使って差分を確認し、必要に応じて **Skill `review`** を使って見落としやリスクを洗い出す  
9. 作業完了後に **Skill `docs`** を使って、README や仕様書などの関連文書を更新する  

## 検証、不具合改修
1. **Skill `plan`** を使って、調査方針・改修方針・確認項目を整理する  
2. **Agent `triager`** を使って、ログ・再現条件・不具合の発生状況を整理する  
3. **Skill `scan`** を使って、原因候補になりそうな関連ファイルや影響範囲を洗い出す  
4. 必要に応じて **Agent `explorer`** を使って、原因箇所周辺の実装や既存処理の流れを調査する  
5. 必要に応じて **Agent `tester`** を使って、再現確認方法や不足している検証観点を整理する  
6. 改修が小さく安全に行える場合は **Skill `refactor`** を使って整理しながら直す  
7. 実際の修正は **Agent `main`** を使って、**Skill `implement`** により改修を行う  
8. 改修後に **Skill `test`** を使って、再発確認・回帰確認・関連機能の検証を行う  
9. 改修後に **Agent `reviewer`** を使って差分を確認し、必要に応じて **Skill `review`** を使って修正漏れや副作用の有無を確認する  
10. 作業完了後に **Skill `docs`** を使って、README や仕様書などの関連文書を更新する
